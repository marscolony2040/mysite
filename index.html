System Architecture

  • Data Pipeline: I stream two websockets from Kraken for information, one socket fetches the orderbooks and the other socket fetches the latest bid/ask prices.
  • Signal Generation: I use a dual signal to enter a trade, one from orderbook anomalies statistical significance and the other from a hidden markov model's classification.
  • Risk Management: My risk management system runs on a function of price volatility which allows my algorithm to have variable exit conditions.
  • Execution Engine: I use Kraken's REST API to execute algorithmic trades. Generally I execute a limit buy order and two limit sell orders in order to minimize fees between trades as opposed to using only market orders.
  • Infrastructure: The backend is built fully in C++ and uses LibTorch to analyze the market with autoencoders. The frontend is built with Tkinter in Python to visualize my trades and pnl. My trades are deposited to an SQL database using Sqlite3 via C++.
